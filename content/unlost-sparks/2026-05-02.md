假如你有几百篇markdown文档，现在需要合并成一份，每篇脚注编码都是使用 1、2、3...，如果直接拼接，那么脚注编号会重复，这会破坏脚注的语义。最好使用全局编号，手动修改当然不太现实，也容易出错，最好把脚注提取出来，统一修改成全局唯一的编号。这里的问题是：

* 如何提取？
* 如何生成全局唯一编号？
* 修改完了如何恢复？

复杂而完善方式当然是是使用一个解析器，比如 Pandoc，修改 AST，生成新的 AST，然后转换、合并。简单的办法呢？可以直接用正则。

我们先来看看 markdown 中最基本的脚注是怎么写的：

```markdown
这里是脚注引用[^1]。

这里再次引用同一个脚注[^1]。

[^1]: Footnote definition
```

脚注有两个基本特点：

* 引用和定义是分离的；
* 同一个脚注可能会有多次引用。

所以我们需要把脚注的定义和引用视为一个整体，关联起来。

我们先建立脚注定义的模型：

```python
from pydantic import BaseModel, Field

class FootnoteDef(BaseChunk):
    placeholder: str = Field(
        default_factory=lambda: f'__PLACEHOLDER_{uuid.uuid4()}__',
    )
    label: str
    content: str
```

这里 `placeholder` 用把脚注所在的文本替换成一个唯一占位符，修改完了以后就可以把修改后的文本恢复到原文中。
`label` 则是对应 `[^1]` 中的 `1` 部分，`content` 则是冒号后面的内容。

脚注引用的模型则会更简单，毕竟它只有一个编号，我们这样设计：

```python
class FootnoteRef(BaseChunk):
    model_config = ConfigDict(frozen=True)
    placeholder: str = Field(
        default_factory=lambda: f'__PLACEHOLDER_{uuid.uuid4()}__',
    )
    label: str
    defined_content: str = ''
```

基本和 `FootnoteDef` 完全一样，`defined_content` 是用来做什么的？引用除了编号还有其他内容吗？我们后面解释它的用处。

接下来用正则来查找文本中的脚注：

```python
FOOTNOTE_DEF_RE = re.compile(
    r'^\[\^(?P<fn_label>.+?)\]:' # match [^label]:
    r'[ \t]*' # followed by optional spaces
    r'(?P<fn_text>.+?)' # followed by any non-empty line
    r'(?=\n|\Z)',
    re.MULTILINE | re.DOTALL
)

FOOTNOTE_REF_RE = re.compile(r'\[\^(?P<fn_label>.+?)\][ \t]*(?!:)')
```

这里的正则没有处理各种edge case，假设你用了最基本的语法，并且脚注定义前后换行。

接下来就是用 `re.sub` 替换，把脚注定义的引用分别替换成各自的占位符，就能获得两个列表：

```python
def parse_footnote_definition(m: re.Match[str]) -> FootnoteDef:
    key = m.group('fn_label')
    content = m.group('fn_text').rstrip()
    # 清理缩进（如果内容有缩进）
    content = re.sub(r'\n[ \t]+', '\n', content)
    fn = FootnoteDef(
        raw=m.group(0),
        label=key,
        content=content,
    )
    # Replace the footenote definition with placeholder
    # so that it won't intefere with the reference search regex.
    return fn

def parse_fn_def(text: str) -> Tuple[str, List[FootnoteDef]]:

    definitions: List[FootnoteDef] = []
    def replace_def(m: re.Match[str]) -> str:
        fn_def = parse_footnote_definition(m)
        definitions.append(fn_def)
        return fn_def.placeholder
        
    text = FOOTNOTE_DEF_RE.sub(replace_def, text)

    return text, definitions

def parse_footnote_reference(m: re.Match[str]):
    return FootnoteRef(
        raw=m.group(0),
        label=m.group('fn_label'),
    )

def parse_fn_ref(text: str) -> Tuple[str, List[FootnoteRef]]:

    references: List[FootnoteRef] = []
    def replace_def(m: re.Match[str]) -> str:
        fn_ref = parse_footnote_reference(m)
        references.append(fn_ref)
        return fn_ref.placeholder
        
    text = FOOTNOTE_DEF_RE.sub(replace_def, text)

    return text, references

example_text = "anything..."

text, definitions = parse_fn_def(example_text)
text, references = parse_fn_ref(text)
```

注意这里的顺序很重要，先替换脚注定义，再解析脚注引用。调换顺序的话，先解析脚注引用会把定义中的 `[^1]` 也吞掉。

我们得到了脚注定义和引用两个独立的列表，然后呢？按照 `label` 字段把它们合并到一个容器中。先定义这个数据模型：

```python
class Footnote(BaseModel):
    definition: FootnoteDef
    references: List[FootnoteRef]

    def set_label(self, label: str):
        self.definition.label = label
        for i, ref in enumerate(self.references):
            self.references[i] = ref.model_copy(
                update={
                    'label': label
                }
            )
```

然后对脚注引用进行分组：

```python
def group_references(references: List[FootnoteRef]) -> Dict[str, List[FootnoteRef]]:
    g = defaultdict(list)
    for ref in references:
        g[ref.label].append(ref)

    return g
```

这里得到的字典，键就是脚注的编号，值是脚注的所有引用。

然后就可以把它们配对：

```python
def pair_footnotes(
    definitions: List[FootnoteDef], 
    references: List[FootnoteRef]
) -> List[Footnote]:
    ref_group = group_references(references)

    return [
        Footnote(
            placeholder=fn.placeholder,
            raw=fn.raw,
            definition=fn,
            references=ref_group.get(fn.label, [])
        )
        for fn in definitions
    ]
```

这样我们就把脚注的定义和引用合并到了一起。接下来就可以按照你的需求对脚注执行任何操作。比如我们开头提到的全局唯一编号，Python 中实现这种简单的自增计数器，有一个非常地道且线程安全的工具 `itertools.count`：

```python
from itertools import count

# 创建一个从 1 开始，每次递增 1 的计数器
global_counter = count(1)

# 每次需要新编号时，直接调用 next()
new_id = next(global_counter) # 1
new_id = next(global_counter) # 2
```

在遍历 `Footenote` 列表时，调用 `set_label` 就可以了。

这个方案还有很多问题，比如代码块中出现了 `[^1]` 结构怎么办？LaTeX 公式块中出现了怎么办？安全一些的方法是采用类似的替换方法，在解析脚注前先把这些关键资产提取出来，替换成唯一占位符，在脚注处理完放回原位后，再把这些代码块、公式块还原回去。

最后解释一下 `defined_content` 字段的用处，有些输出目标的脚注是在原地定义的，而不是分离式，比如 LaTeX、Typest，这时候把让引用携带上脚注的定义比较容易替换，不用再去找它对应的 `FootnoteDef.content` 了。
