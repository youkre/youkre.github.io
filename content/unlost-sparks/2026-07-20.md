这本书的翻译实在让人无语，译者其实可以看懂，但是偏偏翻译起来就开始信口开河，原文的语言很简单，整本书直译就可以做的很不错，可是译者偏要意译，翻译的绕来绕去，丢三落四，甚至干脆自己编，但是自己又表达不清楚。对于数学知识，这些错误累积起来，最后就变得难以理解。

这里以第3章举一些例子。

To see why our current notation can quickly become unwieldy, consider again the gambler’s ruin problem from Chapter 2. In this problem, we may be very interested in how much wealth each gambler has at any particular time. So we could make up notation like letting <span class="math"><math><msub><mi>A</mi><mrow><mi>j</mi><mi>k</mi></mrow></msub></math></span> be the event that gambler A has exactly j dollars after k rounds, and similarly defining an event <span class="math"><math><msub><mi>B</mi><mrow><mi>j</mi><mi>k</mi></mrow></msub></math></span> for gambler B, for all j and k.

原译：

为了认识到当前我们对不确定现象的表达是多么的不明智，首先再次回到第2章赌徒输光的例子。在这个问题中，人们对赌徒在任意时刻还剩多少财产更加感兴趣，因此补充以下表达式：假设事件 <span class="math"><math><msub><mi>A</mi><mrow><mi>j</mi><mi>k</mi></mrow></msub></math></span> 表示赌徒 A 在 k 轮赌博之后恰好剩下 j 美元，类似地，定义事件 <span class="math"><math><msub><mi>B</mi><mrow><mi>j</mi><mi>k</mi></mrow></msub></math></span>。

更正：

我们当前使用的符号很快会变得笨重，要理解原因，可以回顾一下第二章赌徒输光的例子。在这个问题中，我们最感兴趣的可能是每个赌徒在特定时刻还剩多少财产。为此，可以创建符号，例如用 <span class="math"><math><msub><mi>A</mi><mrow><mi>j</mi><mi>k</mi></mrow></msub></math></span> 表示这样的事件：赌徒 A 在 k 轮之后还剩 j 美元，同样定义事件 <span class="math"><math><msub><mi>B</mi><mrow><mi>j</mi><mi>k</mi></mrow></msub></math></span>，表示赌徒 B 在 k 轮后剩余 j 美元。

This is already too complicated. Furthermore, we may also be interested in other quantities, such as the difference in their wealths (gambler A’s minus gambler B’s) after k rounds, or the duration of the game (the number of rounds until one player is bankrupt). Expressing the event “the duration of the game is r rounds” in terms of the <span class="math"><math><msub><mi>A</mi><mrow><mi>j</mi><mi>k</mi></mrow></msub></math></span> and <span class="math"><math><msub><mi>B</mi><mrow><mi>j</mi><mi>k</mi></mrow></msub></math></span> would involve a long, awkward string of unions and intersections. And then what if we want to express gambler A’s wealth as the equivalent amount in euros rather than dollars? We can multiply a number in dollars by a currency exchange rate, but we can’t multiply an event by an exchange rate.

原译：

显然，上述表达过于复杂，更进一步讲,人们也会对其他问题感兴趣，例如在k轮赌博之后两个赌徒所剩财产的差额是多少，或者赌博的持续时间(指其中某个赌徒已经输光)。对于“这局赌博的持续时间”问题，如果用A和B来表示的话，会非常的冗杂。此外，如果想用等价值的欧元替代美元来表示赌徒 A 的财产又将会怎么样?美元可以根据货币汇率乘以一个数字进行转换，然而事件却没有类似的转换比例可以相乘。

更正：

这已经足够复杂了。此外，我们可能还对其他数量感兴趣，例如经过 k 轮后二人剩余财产的差额，或者赌博持续的时间。如果要用 <span class="math"><math><msub><mi>A</mi><mrow><mi>j</mi><mi>k</mi></mrow></msub></math></span> 和 <span class="math"><math><msub><mi>B</mi><mrow><mi>j</mi><mi>k</mi></mrow></msub></math></span> 表达事件“赌博持续了 r 轮”，就要用并集和交集写出来冗长丑陋的字符串。如果要把赌徒A的财产表示成等价值的欧元而非美元，又会怎样？我们可以用汇率乘以一个代表美元的数字，但是没法用汇率乘以一个事件。

Instead of having convoluted notation that obscures how the quantities of interest are related, wouldn’t it be nice if we could say something like the following?

原译：

如果用以下的描述方式来代替上述复杂符号，可能会得到更好的效果。

更正：

复杂的符号会掩盖掉我们感兴趣的数量之间的关系，与其如此，用下面的方式来表述是否更合理？

Thus, a random variable X assigns a numerical value X(s) to each possible outcomes of the experiment.

原译：

因此，一个随机变量 X 为试验的每一个可能的结果，分配数值 X(s)。

解释：

随机变量是一个函数，针对实验的每一种结果，也就是一个元素，这个函数把该元素映射到一个实数。

In writing <span class="math"><math><mi>P</mi><mo stretchy="false">(</mo><mi>X</mi><mo>=</mo><mi>x</mi><mo stretchy="false">)</mo></math></span>, we are using <span class="math"><math><mi>X</mi><mo>=</mo><mi>x</mi></math></span> to denote an event, consisting of all outcomes s to which X assigns the number x. 

原译：

在书写 P(X = x) 时，用 X = x 表示一个事件，它由 X 分配给 x 的所有结果 s 组成。

更正并解释：

写成 <span class="math"><math><mi>P</mi><mo stretchy="false">(</mo><mi>X</mi><mo>=</mo><mi>x</mi><mo stretchy="false">)</mo></math></span> 的形式，我们用 <span class="math"><math><mi>X</mi><mo>=</mo><mi>x</mi></math></span> 表示一个事件(也就是集合)，这个事件由所有的结果 s（即元素）构成，X 则给每一个结果（元素）赋予一个数字 x。
