一个函数的 <span class="math"><math><mi>x</mi></math></span> 可以乘以某个数，<span class="math"><math><mi>y</mi></math></span> 也可以乘以一个数，二者虽然都叫“缩放”，但它们的几何效果、作用方向、甚至“直觉感受”都是根本不同的。
把两者笼统地说成“函数图像被缩放了”是一种**过度简化**，容易造成混淆——尤其是对初学者。下面我们用清晰的方式来梳理两者的本质区别。

## 对 y 方向缩放：y = c · f(x)

对整个函数乘以一个系数（例如，将 <span class="math"><math><mi>f</mi><mo stretchy="false">(</mo><mi>x</mi><mo stretchy="false">)</mo></math></span> 变为 <span class="math"><math><mi>c</mi><mi>f</mi><mo stretchy="false">(</mo><mi>x</mi><mo stretchy="false">)</mo></math></span>，其中<span class="math"><math><mi>c</mi><mo>&gt;</mo><mn>1</mn></math></span>），这相当于在<span class="math"><math><mi>y</mi></math></span>轴方向上的拉伸或压缩。具体来说：
对于 <span class="math"><math><mi>y</mi><mo>=</mo><mi>c</mi><mo lspace="0.2222em" rspace="0.2222em">·</mo><mi>f</mi><mo stretchy="false">(</mo><mi>x</mi><mo stretchy="false">)</mo></math></span> 且 <span class="math"><math><mi>c</mi><mo>&gt;</mo><mn>1</mn></math></span>：函数图像沿着 <span class="math"><math><mi>y</mi></math></span> 轴方向被拉伸，使得所有 <span class="math"><math><mi>y</mi></math></span> 值都相应增大。这意味着图形在垂直方向上变得更高。
对于 <span class="math"><math><mn>0</mn><mo>&lt;</mo><mi>c</mi><mo>&lt;</mo><mn>1</mn></math></span>：则是相反的效果，即图像被压缩，看起来更加扁平。

- 操作对象：输出值（因变量）
- 几何效果：垂直方向拉伸或压缩
- 直观类比：像拉橡皮筋上下拉

规则：

- 若 <span class="math"><math><mi>c</mi><mo>&gt;</mo><mn>1</mn></math></span> → 图像在 <span class="math"><math><mi>y</mi></math></span> 轴方向被拉长（更高、更陡）
- 若 <span class="math"><math><mn>0</mn><mo>&lt;</mo><mi>c</mi><mo>&lt;</mo><mn>1</mn></math></span> → 图像在 <span class="math"><math><mi>y</mi></math></span> 轴方向被压扁（更矮、更平缓）

关键点：<span class="math"><math><mi>x</mi></math></span> 坐标不变，每个点只是“上下移动”

举例：

<span class="math"><math><mi>y</mi><mo>=</mo><mo lspace="0" rspace="0.1667em">sin</mo><mi>x</mi></math></span> vs <span class="math"><math><mi>y</mi><mo>=</mo><mn>2</mn><mo lspace="0.1667em" rspace="0.1667em">sin</mo><mi>x</mi></math></span><br>
→ 波峰从 1 变成 2，但波长（周期）不变。

## 对 x 方向缩放：y = f(kx)

对函数中的 <span class="math"><math><mi>x</mi></math></span> 乘以一个系数（例如，将 <span class="math"><math><mi>f</mi><mo stretchy="false">(</mo><mi>x</mi><mo stretchy="false">)</mo></math></span> 变为 <span class="math"><math><mi>f</mi><mo stretchy="false">(</mo><mi>k</mi><mi>x</mi><mo stretchy="false">)</mo></math></span>，其中<span class="math"><math><mi>k</mi><mo>&gt;</mo><mn>1</mn></math></span>），这相当于在 <span class="math"><math><mi>x</mi></math></span> 轴上进行了一个压缩操作。这意味着，对于相同的 <span class="math"><math><mi>x</mi></math></span> 范围，曲线看起来更加“紧凑”，因为原本分布在更大区间内的特征现在被挤进了更小的空间内。

- 操作对象：输入值（自变量）
- 几何效果：水平方向压缩或拉伸
- 直观类比：像快进/慢放视频 —— 函数“发生”的速度变了

它的规则是反直觉的：

- 若 <span class="math"><math><mi>k</mi><mo>&gt;</mo><mn>1</mn></math></span> → 图像在 <span class="math"><math><mi>x</mi></math></span> 轴方向被压缩（更快达到相同状态）
- 若 <span class="math"><math><mn>0</mn><mo>&lt;</mo><mi>k</mi><mo>&lt;</mo><mn>1</mn></math></span> → 图像在 <span class="math"><math><mi>x</mi></math></span> 轴方向被拉伸（更慢变化）

关键点：<span class="math"><math><mi>y</mi></math></span> 值不变，但达到该 <span class="math"><math><mi>y</mi></math></span> 所需的 <span class="math"><math><mi>x</mi></math></span> 变了

举例：

<span class="math"><math><mi>y</mi><mo>=</mo><mo lspace="0" rspace="0.1667em">sin</mo><mi>x</mi></math></span> vs <span class="math"><math><mi>y</mi><mo>=</mo><mo lspace="0" rspace="0">sin</mo><mo stretchy="false">(</mo><mn>2</mn><mi>x</mi><mo stretchy="false">)</mo></math></span><br>
→ 周期从 <span class="math"><math><mn>2</mn><mi>π</mi></math></span> 缩短为 <span class="math"><math><mi>π</mi></math></span>，图像变“密”了，但振幅仍是 1。

为什么反直觉？
因为我们以为“乘以 2”会让东西变大，但在 <span class="math"><math><mi>f</mi><mo stretchy="false">(</mo><mi>k</mi><mi>x</mi><mo stretchy="false">)</mo></math></span> 中，更大的 <span class="math"><math><mi>k</mi></math></span> 意味着“用更小的 <span class="math"><math><mi>x</mi></math></span> 就能达到原来的输入”，所以图像反而被“挤”到左边。

## 对比总结表

| 变换类型 | 公式 | 效果方向 | <span class="math"><math><mo lspace="0">&gt;</mo><mn>1</mn></math></span> 时效果 | <span class="math"><math><mo lspace="0">&lt;</mo><mn>1</mn></math></span> 时效果 | 是否改变形状？ |
| --- | --- | --- | --- | --- | --- |
| 垂直缩放 | <span class="math"><math><mi>y</mi><mo>=</mo><mi>c</mi><mo lspace="0.2222em" rspace="0.2222em">·</mo><mi>f</mi><mo stretchy="false">(</mo><mi>x</mi><mo stretchy="false">)</mo></math></span> | 上下（<span class="math"><math><mi>y</mi></math></span>） | 拉高、变陡 | 压扁、变平 | 是（斜率变） |
| 水平缩放 | <span class="math"><math><mi>y</mi><mo>=</mo><mi>f</mi><mo stretchy="false">(</mo><mi>k</mi><mi>x</mi><mo stretchy="false">)</mo></math></span> | 左右（<span class="math"><math><mi>x</mi></math></span>） | 压缩（变密） | 拉伸（变疏） | 是（变化率变） |

## 一个帮助记忆的口诀

> “<span class="math"><math><mi>y</mi></math></span> 同向，<span class="math"><math><mi>x</mi></math></span> 反向”
>
> 改 <span class="math"><math><mi>y</mi></math></span>：系数越大，图像越“大”（同向）
>
> 改 <span class="math"><math><mi>x</mi></math></span>：系数越大，图像越“小”（反向压缩）

或者：

> “<span class="math"><math><mi>x</mi></math></span> 的变换是‘逆时间’的”
>
> <span class="math"><math><mi>f</mi><mo stretchy="false">(</mo><mn>2</mn><mi>x</mi><mo stretchy="false">)</mo></math></span> 就像把原函数“快放 2 倍”，所以图像在水平方向“缩短”了。

## 加减呢？

平移（translation）中的符号方向（<span class="math"><math><mi>x</mi><mo>+</mo><mi>a</mi></math></span> 与 <span class="math"><math><mi>x</mi><mo>−</mo><mi>a</mi></math></span>）和缩放一样，也是“反直觉”的经典案例。靠死记硬背很容易：“左加右减”，但我们需要理解为什么。

函数 $ y = f(x - a) $ 的图像，是将 $ y = f(x) $ 向右平移 <span class="math"><math><mi>a</mi></math></span> 个单位。<br>
但为什么是减号却向右移？这看起来完全相反！

同样：

- $ f(x + a) $ → 向左平移 <span class="math"><math><mi>a</mi></math></span>
- $ f(x - a) $ → 向右平移 <span class="math"><math><mi>a</mi></math></span>

> ❓ 直觉上会觉得：“+ 应该往右，– 应该往左”，但事实却相反。

### 正确理解方式：“补偿视角”（Compensation View）

想象你是一个输入检查员，负责把 <span class="math"><math><mi>x</mi></math></span> 值交给函数 <span class="math"><math><mi>f</mi></math></span>。

- 原函数 <span class="math"><math><mi>y</mi><mo>=</mo><mi>f</mi><mo stretchy="false">(</mo><mi>x</mi><mo stretchy="false">)</mo></math></span>：你直接把 <span class="math"><math><mi>x</mi></math></span> 给 <span class="math"><math><mi>f</mi></math></span>。
- 新函数 <span class="math"><math><mi>y</mi><mo>=</mo><mi>f</mi><mo stretchy="false">(</mo><mi>x</mi><mo>−</mo><mi>a</mi><mo stretchy="false">)</mo></math></span>：你必须先把 <span class="math"><math><mi>x</mi></math></span> 减去 <span class="math"><math><mi>a</mi></math></span>，再交给 <span class="math"><math><mi>f</mi></math></span>。

那么，要让新函数在某个位置“表现得和原函数在原点一样”，你需要怎么做？

### 举个具体例子

设 <span class="math"><math><mi>f</mi><mo stretchy="false">(</mo><mi>x</mi><mo stretchy="false">)</mo><mo>=</mo><msup><mi>x</mi><mn>2</mn></msup></math></span>，顶点在 <span class="math"><math><mi>x</mi><mo>=</mo><mn>0</mn></math></span>。

现在看 <span class="math"><math><mi>g</mi><mo stretchy="false">(</mo><mi>x</mi><mo stretchy="false">)</mo><mo>=</mo><mi>f</mi><mo stretchy="false">(</mo><mi>x</mi><mo>−</mo><mn>2</mn><mo stretchy="false">)</mo><mo>=</mo><mo stretchy="false">(</mo><mi>x</mi><mo>−</mo><mn>2</mn><msup><mo stretchy="false">)</mo><mn>2</mn></msup></math></span>。

问：<span class="math"><math><mi>g</mi><mo stretchy="false">(</mo><mi>x</mi><mo stretchy="false">)</mo></math></span> 的顶点在哪里？

解：当 <span class="math"><math><mi>x</mi><mo>−</mo><mn>2</mn><mo>=</mo><mn>0</mn></math></span> 时，<span class="math"><math><mi>g</mi><mo stretchy="false">(</mo><mi>x</mi><mo stretchy="false">)</mo><mo>=</mo><mn>0</mn></math></span> → 所以 <span class="math"><math><mi>x</mi><mo>=</mo><mn>2</mn></math></span>。

> 💡 为了让 <span class="math"><math><mi>f</mi></math></span> “看到” 0（从而输出最小值），你必须给它一个比 0 大 2 的输入（即 <span class="math"><math><mi>x</mi><mo>=</mo><mn>2</mn></math></span>）。<br>
> 换句话说：整个图形必须向右移动 2 个单位，才能让 <span class="math"><math><mi>f</mi></math></span> 在新位置“重现”原来的行为。

### 口诀升级版

> “函数总是忠于它的输入。你想让它在新地方做旧事，就得提前‘补偿’。”

- 想让图像 **向右移 <span class="math"><math><mi>a</mi></math></span>**？<br>
  → 那么在每个 <span class="math"><math><mi>x</mi></math></span> 处，都要**假装自己在 <span class="math"><math><mi>x</mi><mo>−</mo><mi>a</mi></math></span>**（因为右边的点要模仿左边的行为）<br>
  → 所以写成 <span class="math"><math><mi>f</mi><mo stretchy="false">(</mo><mi>x</mi><mo>−</mo><mi>a</mi><mo stretchy="false">)</mo></math></span>

- 想让图像 **向左移 <span class="math"><math><mi>a</mi></math></span>**？<br>
  → 在每个 <span class="math"><math><mi>x</mi></math></span> 处，都要**假装自己在 <span class="math"><math><mi>x</mi><mo>+</mo><mi>a</mi></math></span>**<br>
  → 所以写成 <span class="math"><math><mi>f</mi><mo stretchy="false">(</mo><mi>x</mi><mo>+</mo><mi>a</mi><mo stretchy="false">)</mo></math></span>

### 🆚 对比总结：平移 vs 缩放

| 变换 | 表达式 | 实际效果 | 为什么“反直觉”？ |
|------|--------|---------|----------------|
| 水平平移 | <span class="math"><math><mi>f</mi><mo stretchy="false">(</mo><mi>x</mi><mo>−</mo><mi>a</mi><mo stretchy="false">)</mo></math></span> | 向右 移 <span class="math"><math><mi>a</mi></math></span> | 函数需要“提前看到”未来的输入 |
| 水平缩放 | <span class="math"><math><mi>f</mi><mo stretchy="false">(</mo><mi>k</mi><mi>x</mi><mo stretchy="false">)</mo><mo>,</mo><mtext> </mtext><mi>k</mi><mo>&gt;</mo><mn>1</mn></math></span> | 向左压缩 | 函数用更小的 <span class="math"><math><mi>x</mi></math></span> 就达到原效果 |
| 垂直平移 | <span class="math"><math><mi>f</mi><mo stretchy="false">(</mo><mi>x</mi><mo stretchy="false">)</mo><mo>+</mo><mi>b</mi></math></span> | 向上移 <span class="math"><math><mi>b</mi></math></span> | 同向！不反直觉 |
| 垂直缩放 | <span class="math"><math><mi>c</mi><mi>f</mi><mo stretchy="false">(</mo><mi>x</mi><mo stretchy="false">)</mo><mo>,</mo><mtext> </mtext><mi>c</mi><mo>&gt;</mo><mn>1</mn></math></span> | 向 上拉伸 | 同向！不反直觉 |

> 只有对 <span class="math"><math><mi>x</mi></math></span> 的操作（无论是加减还是乘除）才是“反向”的，因为你在操控输入，而不是输出。

### 💡 记忆技巧（终极版）

> “对 <span class="math"><math><mi>x</mi></math></span> 做什么，图像就朝相反方向跑。”

- <span class="math"><math><mi>x</mi><mo stretchy="false">→</mo><mi>x</mi><mo>+</mo><mi>a</mi></math></span>（加）→ 图像 左移
- <span class="math"><math><mi>x</mi><mo stretchy="false">→</mo><mi>x</mi><mo>−</mo><mi>a</mi></math></span>（减）→ 图像 右移
- <span class="math"><math><mi>x</mi><mo stretchy="false">→</mo><mn>2</mn><mi>x</mi></math></span>（乘 >1）→ 图像 压缩（左挤）
- <span class="math"><math><mi>x</mi><mo stretchy="false">→</mo><mi>x</mi><mo lspace="0" rspace="0">/</mo><mn>2</mn></math></span>（除 >1）→ 图像 拉伸（右扩）

而对 <span class="math"><math><mi>y</mi></math></span> 的操作（即整个函数）：
> “对 <span class="math"><math><mi>y</mi></math></span> 做什么，图像就朝相同方向走。”

- <span class="math"><math><mi>y</mi><mo stretchy="false">→</mo><mi>y</mi><mo>+</mo><mi>b</mi></math></span> → 上移
- <span class="math"><math><mi>y</mi><mo stretchy="false">→</mo><mn>2</mn><mi>y</mi></math></span> → 拉高
