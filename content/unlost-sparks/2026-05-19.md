在学习逻辑回归或神经网络时，我们经常会看到这样两个公式：

<div class="math">
<math display="block"><mi>z</mi><mo>=</mo><mi>𝐰</mi><mo lspace="0.2222em" rspace="0.2222em">·</mo><mi>𝐱</mi><mo>+</mo><mi>b</mi></math>
</div>

<div class="math">
<math display="block"><mi>p</mi><mo>=</mo><mi>σ</mi><mo stretchy="false">(</mo><mi>z</mi><mo stretchy="false">)</mo><mo>=</mo><mfrac><mn>1</mn><mrow><mn>1</mn><mo>+</mo><msup><mi>e</mi><mrow><mo lspace="0" rspace="0">−</mo><mi>z</mi></mrow></msup></mrow></mfrac></math>
</div>

书上通常会轻描淡写地说：“<span class="math"><math><mi>z</mi></math></span> 是 Sigmoid 函数的输入，也叫 Logit。”

这时候，初学者很容易产生一种错觉：仿佛 <span class="math"><math><mi>z</mi></math></span> 这个数字天生就带着某种神秘的魔力，注定要经过 Sigmoid 变换。

其实不然。 让我们剥开数学的外衣，看看这里面到底发生了什么。

## 两个独立存在的“老古董”函数

在机器学习诞生之前，数学界早就定义了两对互逆的函数，用来处理概率问题：

Logit 函数（对数几率），定义是 

<div class="math">
<math display="block"><mo lspace="0" rspace="0">ln</mo><mo stretchy="false">(</mo><mfrac><mi>p</mi><mrow><mn>1</mn><mo>−</mo><mi>p</mi></mrow></mfrac><mo stretchy="false">)</mo></math>
</div>

它的作用是把一个在 <span class="math"><math><mo stretchy="false">(</mo><mn>0</mn><mo>,</mo><mn>1</mn><mo stretchy="false">)</mo></math></span> 之间的概率 <span class="math"><math><mi>p</mi></math></span>，强行“拉伸”到整个实数轴 <span class="math"><math><mo stretchy="false">(</mo><mo lspace="0" rspace="0">−</mo><mi>∞</mi><mo>,</mo><mo lspace="0" rspace="0">+</mo><mi>∞</mi><mo stretchy="false">)</mo></math></span>。

Sigmoid 函数（Logistic 函数），它的定义是

<div class="math">
<math display="block"><mfrac><mn>1</mn><mrow><mn>1</mn><mo>+</mo><msup><mi>e</mi><mrow><mo lspace="0" rspace="0">−</mo><mi>z</mi></mrow></msup></mrow></mfrac></math>
</div>

作用是把一个在 <span class="math"><math><mo stretchy="false">(</mo><mo lspace="0" rspace="0">−</mo><mi>∞</mi><mo>,</mo><mo lspace="0" rspace="0">+</mo><mi>∞</mi><mo stretchy="false">)</mo></math></span> 之间的任意实数 <span class="math"><math><mi>z</mi></math></span>，强行“压缩”回 <span class="math"><math><mo stretchy="false">(</mo><mn>0</mn><mo>,</mo><mn>1</mn><mo stretchy="false">)</mo></math></span> 之间。

这两个函数在数学字典里是“死”的定义，它们和机器学习原本没有任何关系。

## 机器学习遇到的“麻烦”

当我们想用计算机做分类（比如判断一封邮件是不是垃圾邮件）时，面临一个难题：

我们手里有什么？ 我们有一堆特征（比如邮件里的单词），通过线性计算 <span class="math"><math><mi>𝐰</mi><mo lspace="0.2222em" rspace="0.2222em">·</mo><mi>𝐱</mi><mo>+</mo><mi>b</mi></math></span>，我们得到了一个原始得分 <span class="math"><math><mi>z</mi></math></span>。这个 <span class="math"><math><mi>z</mi></math></span> 可正可负，可大可小，没有任何限制。

我们想要什么？ 我们想要一个概率 <span class="math"><math><mi>p</mi></math></span>。比如“这是垃圾邮件的概率是 80%”。概率必须在 <span class="math"><math><mo stretchy="false">(</mo><mn>0</mn><mo>,</mo><mn>1</mn><mo stretchy="false">)</mo></math></span> 之间。

矛盾出现了，怎么把那个无拘无束的 <span class="math"><math><mi>z</mi></math></span>，变成规规矩矩的概率 <span class="math"><math><mi>p</mi></math></span>？

## “借壳”上市：强行建立关系

为了解决这个矛盾，逻辑回归做了一个非常聪明（也可以说是“投机取巧”）的操作：

我们“假设”那个原始得分 <span class="math"><math><mi>z</mi></math></span>，恰好就是概率 <span class="math"><math><mi>p</mi></math></span> 的 Logit 值。

换句话说，我们强行规定：

<div class="math">
<math display="block"><mi>z</mi><mo>=</mo><mtext>logit</mtext><mo stretchy="false">(</mo><mi>p</mi><mo stretchy="false">)</mo><mo>=</mo><mo lspace="0" rspace="0">ln</mo><mo stretchy="false">(</mo><mfrac><mi>p</mi><mrow><mn>1</mn><mo>−</mo><mi>p</mi></mrow></mfrac><mo stretchy="false">)</mo></math>
</div>

这就好比说：“嘿，<span class="math"><math><mi>z</mi></math></span>，虽然你是线性算出来的，但从现在开始，我就把你当成是某个概率的对数几率来用。”

既然我们把 <span class="math"><math><mi>z</mi></math></span> 定义为了 Logit，那么根据数学定义，要求出原本的概率 <span class="math"><math><mi>p</mi></math></span>，自然就要用 Logit 的反函数——也就是 Sigmoid。

## 所以，真相是这样的

回顾一下这个过程，你就会发现 <span class="math"><math><mi>z</mi></math></span> 本身并没有什么特殊的魔力：

1. 计算： 模型算出一个原始分数 <span class="math"><math><mi>z</mi></math></span>。
2. 解释： 我们借用数学概念，说“这个 <span class="math"><math><mi>z</mi></math></span> 代表了对数几率（Logit）”。
3. 转换： 既然是对数几率，我们就用 Sigmoid 函数把它反转回概率 <span class="math"><math><mi>p</mi></math></span>。

## 总结

Logit 是我们给原始得分 <span class="math"><math><mi>z</mi></math></span> 起的 “数学外号” ，而 Sigmoid 是我们为了把这个外号还原成概率而使用的 “翻译工具” 。

下次当你看到代码里的变量名叫 `logits` 时，你就知道：它指的就是那个还没经过 Sigmoid 处理的、最原始的“裸分”。
